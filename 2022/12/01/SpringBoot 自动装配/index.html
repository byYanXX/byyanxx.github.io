SpringBoot 自动装配原理

什么是 SpringBoot 自动装配

没有 SpringBoot 时，即使有注解的帮助，还是需要手动编写 ApplicationContext.xml 配置文件。有了 SpringBoot 可以通过 application.properties 和 application.yml 来实现全局配置文件。

没有 Spring Boot 的情况下，如果我们需要引入第三方依赖，需要手动配置，非常麻烦。Spring Boot 中，我们直接引入一个 starter 即可。

比如，如果想使用 Redis，直接在项目中引入对应的 starter 即可：

引入 starter 之后，我们通过少量注解和一些简单的配置就能使用第三方组件提供的功能了。

所谓自动装配，就是通过注解或者一些简单的配置就能在 Spring Boot 的帮助下实现某块功能。

如何实现自动装配的？

先看核心注解 @SpringBootApplication，该注解可以看做是 @Configuration、@EnableAutoConfiguration、@ComponentScan 注解的集合。其中 @EnableAutoConfiguration 表示的是「启用 SpringBoot 的自动配置机制」。

所以从 EnableAutoConfiguration 这个注解入手。

思路：

@SpringBootApplication —> @EnableAutoConfiguration —> AutoConfigurationImportSelector 类 —> selectImports() 方法 —> getAutoConfigurationEntry() 方法，该方法即为自动装配的实现

自动装配步骤

  1. 判断自动装配开关是否打开，默认 spring.boot.enableautoconfiguration=true

  2. 获取 @EnableAutoConfiguration 注解中的 exclude 和 excludeName

  3. 获取需要自动装配的所有配置类，读取 META-INF/spring.factories

    不光是这个依赖下的 META-INF/spring.factories 被读取到，所有 Spring Boot Starter 下的 META-INF/spring.factories 都会被读取到

  4. spring.factories 中这么多配置，不可能每次启动都要全部加载，所以再次筛选，只有 @ConditionalOnXXX 中的所有条件都满足，该类才会生效

小结

  • Spring Boot 通过 @EnableAutoConfiguration 开启自动装配

  • 通过 SpringFactoriesLoader 最终加载 META-INF/spring.factories 中的「自动配置类」实现自动装配

    • 自动配置类就是通过 @Conditional 按需加载的配置类，想要其生效必须引入 spring-boot-starter-xxx 包实现起步依赖。
